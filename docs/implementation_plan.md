# Kikillionaire 가계부 웹앱 개발 계획 (최종)

워킹홀리데이 기간 동안 자산을 체계적으로 관리하고 생존(Runway) 기간을 늘리기 위한 나만의 가계부 앱(Kikillionaire)을 개발합니다. Next.js를 기반으로 구축하며, 데이터베이스로 **구글 시트(Google Sheets)**를 사용하고 스타일링은 **바닐라 CSS**를 사용합니다.

## User Review Required

> [!IMPORTANT]
> **구글 시트 연동 방식 (Google Apps Script API)**
> Supabase 대신 구글 시트를 사용하기 위해, 구글 시트에 내장된 **Google Apps Script(GAS)를 웹 앱으로 배포**하여 Next.js가 데이터를 읽고 쓸 수 있는 API 엔드포인트로 활용합니다. 
> 개발 과정에서 구글 시트에 붙여넣을 간단한 Apps Script 코드와 시트 헤더(컬럼) 구성을 친절히 안내해 드릴 예정입니다.

> [!NOTE]
> **Make.com 및 이메일 파싱 데이터 구성**
> Make.com에서 Gmail을 파싱해 구글 시트에 추가해야 하므로, 본 웹앱이 정상 작동하려면 구글 시트가 특정 형식의 컬럼을 가지고 있어야 합니다. 
> **필요한 시트 컬럼 설계**:
> 1. `Transactions` 시트: 일자, 카테고리(수익/지출/이체), 출발자산(출처), 도착자산(이체 시), 금액, 통화(KRW/EUR), 가맹점명, 메모
> 2. `Assets` 시트: 자산명, 유형(예금/적금/주식/현금), 현재잔액, 통화(KRW/EUR), 이자율/티커(적금/주식용)

## Open Questions & API 구성

- **환율 API**: 무료 환율 API인 `https://open.er-api.com/v6/latest/EUR`를 활용해 EUR/KRW 환율을 실시간으로 가져옵니다.
- **주식 API**: 한국 및 독일/해외 주식을 조회할 수 있도록 야후 파이낸스(Yahoo Finance)의 무료 비공식 API 혹은 국외 주식 조회가 가능한 공인 무료 API를 Next.js Route Handler(백엔드 API)에 연동해 실시간 시세를 반영합니다. (수동 시세 업데이트 기능도 병행 제공)

## Proposed Changes

### 1. 전역 디자인 시스템 및 레이아웃
- **`styles/variables.css`**: 색상, 여백, 서체, 텍스트 크기 등을 정의한 전역 CSS 변수 파일 생성.
- **`styles/globals.css`**: 기본 여백 제거, 반응형 기본 레이아웃 구성. 그림자나 애니메이션을 최소화하여 편집하기 편하게 구성.
- **`components/ui/...`**: 재사용 가능한 전역 컴포넌트 (`ToggleSwitch.tsx`, `Card.tsx`, `Modal.tsx`, `Button.tsx` 등).

### 2. 자산 관리 (Asset Management)
- **자산 그룹화**:
  - **가용 자산**: 한국 예금, 트래블월렛(EUR), N26(EUR), 현금(EUR)
  - **비가용 자산**: 한국 적금, 한국 주식 (적금은 일할 이자 계산기 작동, 주식은 실시간 API 또는 수동 입력 단가 기반 평가금 계산)
- **환율 토글 기능**: 우측 상단 토글 스위치를 통해 전체 자산과 각 항목을 원화(KRW) 또는 유로(EUR)로 실시간 변환하여 표시.

### 3. 거래 내역 관리 및 이중 계산 방지
- **카테고리 분리 (이중 계산 방지 로직 핵심)**
  - 수익(Income), 지출(Expense) 외에 **이체(Transfer)** 분류를 사용합니다.
  - *사례 1 (계좌 이체)*: 한국 계좌에서 트래블월렛으로 돈을 충전할 경우 `Transfer`로 기록. 총 자산은 변하지 않고, 예금 잔액이 줄고 트래블월렛 잔액이 늡니다.
  - *사례 2 (현금 인출)*: N26 계좌에서 ATM으로 유로를 인출할 경우 역시 `Transfer`로 기록하여 현금 자산을 늘리고 계좌 잔액을 줄입니다.
  - 실제 카드결제나 현금을 소비할 때만 `Expense`로 기록되도록 하여 이중 계산을 완벽히 방지합니다.
- **수동 입력 지원**: 현금 사용 내역이나 주식 변동 등 이메일 알림이 없는 항목은 직접 입력 가능한 UI를 구축합니다.

### 4. 대시보드 및 생존 인사이트 (Insights)
- **예산 대비 지출 현황**: 1개월 목표 예산 설정 기능을 두고, 현재 지출이 예산을 초과했는지 시각적인 바 차트(Progress Bar)로 경고.
- **생존 기간 (Runway) 계산**:
  - `(남은 가용 자산) ÷ (최근 1개월 평균 지출액 - 월 평균 수익액) = 버틸 수 있는 개월 수`
  - 이를 통해 돈이 언제 바닥나는지 직관적으로 알려줍니다.
- **일일 권장 지출액**: 남은 월 예산을 남은 일수로 나누어 "오늘 하루 얼마까지만 써야 하는지"를 가이드합니다.

## Verification Plan

### Manual Verification
1. `npx create-next-app`으로 프로젝트 초기화 후 로컬 개발 서버(`npm run dev`) 구동하여 전역 변수(CSS)가 정상 적용되는지 확인.
2. 메인 대시보드에서 유로/원화 토글 스위치를 눌렀을 때 자산 총액 및 개별 항목이 환율에 맞게 즉시 변환되는지 테스트.
3. 이체(Transfer) 기능 테스트: 트래블월렛 충전 시 자산 총액은 그대로 유지되며 두 계좌 잔액만 바뀌는지 검증.
4. 인사이트 테스트: 가상의 수입/지출 데이터를 넣어 생존 기간(Runway)과 목표 초과 여부가 정확하게 계산되어 표시되는지 확인.
